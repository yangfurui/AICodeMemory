# v0.1.0(2026-07-08)

## 概述

首版跑通全链路:`cmem index / search / status` 三命令,把 `~/.claude/projects/` 的会话历史索引成本地语义检索库。453 会话 → 13,265 块,首次索引 274s(含模型下载),增量重跑 12s,检索质量 5 问 4 中 top-1。

## 已确认的机制(含证伪记录)

- **bge-small-zh-v1.5 输出 512 维,不是 384**。英文版 small 才是 384,首次全量后 `load_matrix` reshape 报错暴露;按排查纪律先验证假设(`get_sentence_embedding_dimension()` 实测 512)再修。库中数据无损(存的就是模型真实输出),只改 DIM 常量,未重建。
- **切块上限必须对齐模型 512 token**:超限部分嵌入时被静默截断、检索不到。450 字符上限实测 p99=450、0 块超 600。
- **增量防重(sid+mtime)精确工作**:重跑仅重索引 mtime 变化的会话(实测 453 个中只有正在进行的当前会话被重索引)。
- **自指污染(已知课题,未修)**:讨论历史的新会话会与历史本身竞争排名("40006"一问 top-1 被当天引用它的会话占据)。候选方案:时间衰减/排除会话/`--before`,留 v0.2 评估。

## 改动列表

- extractor:jsonl 解析与去噪(只留 user/assistant 文本;滤 thinking/tool_use/tool_result/sidechain/系统噪音/斜杠命令样板;同角色连续消息合并)
- chunker:Q+A 一块,超长按段落装箱拆块且每块带问题前缀(≤120 字)
- embedder:bge-small-zh-v1.5 本地推理,查询侧加官方指令前缀,归一化输出
- store:SQLite 单文件(chunks/processed/meta 三表);确定性块 ID upsert;schema/模型版本变更自动作废重建
- searcher:全库精确 cosine(numpy)→ top-50 候选 → jieba BM25 候选内重排 → 0.6/0.4 融合
- cli:index(增量,惰性加载模型)/ search / status
