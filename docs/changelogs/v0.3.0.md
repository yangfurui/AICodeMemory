# v0.3.0(2026-07-08)— 数据安全版(档案化)

## 概述

产品定位根本修正:库不是"可随意重建的缓存",是**档案**——源(`~/.claude/projects/`)被 Claude Code 按 30 天滚动清理,库中 30 天外的内容是唯一副本。本版新增 raw 原始层(源文件 gzip 底片)、拆掉"升级即全清"的雷、用契约测试锁死"永不丢数据"。产品叙事定型:**Claude Code 只给你 30 天,AICodeMemory 把它变成永久。**

## 已确认的机制(含证伪记录)

- **"库=纯派生缓存"的旧定位被用户质疑并证伪**:该叙事隐含"源永在",而源 30 天即清;v0.2 的 `_check_versions` 在版本不匹配时 `DELETE FROM chunks` 全清重灌,等于每次升级销毁 30 天前的记忆(v0.2 升级已发生一次,未损失纯因当时源均在窗口内)。
- **三层生命周期模型**:raw 底片(gzip 源,永不自动删)/ text 剪报(对话原文,永不自动删)/ 索引(向量+FTS,随时可从 text 重算)。"永不全清"成为最强不变量——代码中不再存在对 chunks 的全表删除。
- **双源索引**:每次 index 遍历"现存源 + raw 存档",账本(sid+mtime)天然去重;由此 fresh 重建、换机迁移、源消失三个场景自动统一(raw 即历史)。
- **真机验证遗忘对抗**:移走真实源文件(96 块的会话)→ index → 96→96 块、raw 底片在、过滤检索 top-1 命中原话。
- 实测压缩率:241MB 源 → 93.4MB gzip(约 2.6:1),478 份 49.5s。
