# AICodeMemory v0.3.0 方案 — 数据安全版(档案化)

> 2026-07-08 起草。用户裁决:"原始数据肯定是要保留的,不管它放在哪里"。
> 原定 v0.3(MCP 集成)顺延为 v0.4——数据安全优先于功能。
>
> **2026-07-14 契约勘误**:本方案原把 `chunks.text` 也定义成永不替换的档案,
> 与实际的按会话事务替换相矛盾。最终契约收敛为:**raw 是永久事实来源,
> text/chunks 是可从 raw 重建的当前检索投影**。

## 定位修正(本版的根本动因)

v0.1/v0.2 把全部本地数据定义为"派生缓存,可随时从上游源重建,不用备份"。**该定位对普通用户是错的**:源(`~/.claude/projects/`)被 Claude Code 按 30 天滚动清理,超过窗口的 raw 可能是全世界唯一副本——**raw 是档案,SQLite 是由它派生的检索投影**。

由此暴露的雷:`store._check_versions` 在 schema/模型变更时 `DELETE FROM chunks` 全清重灌,而重灌只能覆盖存活的源 → **用户每次升级,30 天前的记忆被销毁**(v0.2 升级已发生一次,未造成损失纯因用户源均在窗口内)。

## 数据模型:三层生命周期

| 层 | 内容 | 生命周期 | 再生性 |
|---|---|---|---|
| **raw 永久档案**(新增) | 源 jsonl 的 gzip 拷贝,`~/.cmem/raw/<项目>/<会话>.jsonl.gz` | **永不自动删除**;非追加改写拒绝覆盖 | 不可再生(底片) |
| **text 当前投影** | chunks.text 去噪/切块文本 | 按会话原子替换 | 可从 raw 重新提取 |
| **索引层** | embedding、chunks_fts | 随时可弃 | 可从 text 投影重算 |

## 技术方案

### A. raw 原始层

- `cmem index` 处理每个"新/有变化"的会话时,先把源 jsonl gzip 压缩存入 raw 层;mtime + size 指纹未变则跳过;
- 新源只有以旧底片完整字节内容为前缀时才能覆盖更新;截断、改写或旧底片无法验证时显式失败;
- 源文件从磁盘消失 ≠ 删除其 raw 存档(append-only);
- 空间预估:实测源 241MB/月,jsonl 结构重复度高,gzip 后约 30-50MB/月,一年 <600MB,可接受;
- raw 层的价值:任何未来的去噪/切块算法改进,都能回溯应用到全部历史(含已过期源)。

### B. 拆雷:版本轴拆分,raw 永不因升级丢失

把单一 `SCHEMA_VERSION` 拆成两根独立版本轴:

- **`EXTRACT_VERSION`**(去噪/切块算法)变更 → 从 **raw 层**逐会话原子替换 text 当前投影 → 重算索引;
- **`MODEL`**(embedding 模型)变更 → text 投影一字不动,仅从库内 text 重算向量;
- 纯表结构变更 → SQL 迁移,不触碰数据。

任何路径下,**raw 永不删除;text 只在单会话新投影成功落库时被原子替换**。

### C. 契约固化

三类不变量写成测试,防未来改动违约:

1. 源文件消失后运行 `cmem index` → 该会话的块与 raw 存档完整保留、可检索;
2. 源截断/改写 → 拒绝覆盖旧 raw;投影替换失败 → 旧 chunks/FTS/账本整体回滚;
3. 模拟模型/算法版本变更 → 模型变更不改 text;同一 raw 能重建等价 text 投影。

### D. 体检与备份指引

- `cmem status` 增加:raw 层规模(文件数/大小)、`PRAGMA integrity_check` 结果;
- README 定位段与备份建议反转:**`~/.cmem/` 整个目录(库+raw)包含超过 30 天窗口的唯一记录,应纳入用户备份**(Time Machine/云盘);产品叙事改为"Claude Code 只给你 30 天,AICodeMemory 把它变成永久"。

## 验收标准

1. **遗忘对抗**:把某个已索引会话的源文件移走(模拟 30 天清理)→ `cmem index` → 块仍在、检索正常、raw 存档仍在;
2. **升级可重建**:模拟 MODEL 变更触发重算 → text/raw 不变;模拟 EXTRACT_VERSION 变更 → 从 raw 重建等价当前投影(含已"消失"源的会话);
3. **raw 增量与改写保护**:二次 index 秒级跳过未变化会话;追加内容安全更新;截断/改写不覆盖旧底片;
4. **回归**:五连测不回退;status 显示 raw 规模与 integrity ok。

## 遗留课题(v0.4 候选,2026-07-09 记)

- **`cmem verify`**:遍历 raw 逐文件解压验 gzip CRC(478 份约几秒),把"损坏能发现"从被动变主动;与备份配合构成档案保全闭环。
- **`cmem show <session>` 上下文展开**:search 命中块只有 450 字,真实需求常是"看这次对话的全景"。从 text 当前投影按 session_id + chunk_index 取相邻块(毫秒级,有索引),可选 `--raw` 按需解压单份底片看未去噪原文(几十 ms)。原则:**检索热路径永不碰 raw**,raw 只做单文件按需读取。
