# claudeMemory v0.1.0 方案

> 2026-07-08 与 Claude 对话中逐条确认,此文档为确认结果的固化。

## 定位

给 Claude Code 的本地会话记忆检索工具:把 `~/.claude/projects/` 下的历史会话索引成可语义检索的本地库,让人和 AI 都能找回"当时的原话"。独立开源项目(GitHub),与任何私有工作流解耦。

**卖点(差异化,对照 MemPalace):**

- 极简:核心几百行、单文件 SQLite、零服务、零守护进程、零配置
- 中文一等公民:bge 中文 embedding + jieba 分词 BM25(MemPalace 无中文分词)
- 数据不出机,无 API key

## 已确认的技术决策

| 决策点 | 结论 | 理由 |
|---|---|---|
| 语言 | Python(标准 venv + pip,不用 uv) | embedding 生态;机器无 uv,不引新工具 |
| 数据源 | 默认 `~/.claude/projects/`,仅留 `--source` 参数覆盖 | 所有用户开箱即用;零配置文件 |
| 存储 | 单文件 SQLite,向量存 BLOB | 个人量级(万~几十万块)无需向量库 |
| 检索 | numpy 全量精确 cosine,**不用 ANN/ChromaDB** | 10 万块内毫秒~百毫秒级;精确>近似;零索引维护。查询>500ms 时再换引擎(sqlite-vec/FAISS),数据格式不变 |
| embedding | `BAAI/bge-small-zh-v1.5`(~100MB,**512 维**——中文版与英文版 small(384)维度不同,已实测踩坑) | 中文最优小模型;HF 直连已验证通畅 |
| 混合权重 | 0.6×向量 + 0.4×BM25(jieba 分词) | MemPalace 验证过的配比,起步值 |
| 切块 | 一问一答为一块,超长按 ~600 字再切 | 保留问答上下文的最小完整单元 |
| 幂等 | 块 ID = hash(session_id+序号) 确定性 upsert;processed 表记 session mtime | 重跑无重复;有新内容自动重索引(借鉴 trace 的 mtime 模式) |
| AI 集成 | 无 MCP、无服务——用户 CLAUDE.md 加一条"查历史用 cmem search" | Claude Code 本身会跑 Bash |
| 库文件定位 | 派生缓存,可随时从源全量重建,不备份不入 git | 源(jsonl)是唯一真相 |
| License | MIT | — |

## 架构

```
~/.claude/projects/**/*.jsonl
   │  cmem index(手动/用户自行定时)
   ▼
extractor(去噪:只留人/AI 文本,滤工具输出、sidechain、系统噪音)
   → chunker(Q+A 配对切块)
   → embedder(bge 批量嵌入)
   → store(sqlite:chunks + processed 两张表)
   ▼
~/.claudememory/memory.sqlite3
   ▲
   │  cmem search "问题"
searcher(查询嵌入 → numpy cosine → jieba BM25 → 6:4 融合 → top-5 带出处)
```

## CLI

- `cmem index` — 增量索引(首次即全量)
- `cmem search "<问题>" [-k 5]` — 检索,输出相似度/日期/项目/会话/原文块
- `cmem status` — 库概况(块数、会话数、日期覆盖)

## v0.1 明确不做

MCP server、自动钩子、配置文件、多 embedding 模型、导出、知识图谱、AAAK 类压缩、Web UI。

## 已知课题(v0.1 不修,留给后续版本)

- **自指污染**:讨论历史的新会话被索引后,会与"历史本身"竞争排名(实测"40006 根因"的 top-1 被当天引用它的会话占据)。候选方案:检索时按时间衰减加权 / 支持排除指定会话 / `--before` 日期过滤。
- BM25 权重 0.4 下,高频复述内容(如日记生成会话)易获高分,可能需要按项目降权。
