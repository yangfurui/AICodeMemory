# AICodeMemory v0.4.0 方案 — MCP 集成版

> 2026-07-09 起草。核心:一个 MCP server 让 Claude Code / Claude Desktop / Codex / Cursor
> 全都能原生调用记忆;`cmem setup` 把集成收敛为一条命令。
> 2026-07-14 开始按原子任务逐项交付,任务 1 已完成。

## 动因

- 现状集成靠 CLAUDE.md 软约定 + Bash:只覆盖 Claude Code,调用靠模型"记得",每次冷启动 3-4s;
- MCP 化后:工具带 schema 正式注册(调用意愿与准确性显著提升)、跨客户端通吃、
  **server 常驻使模型与矩阵只加载一次——查询从 3-4s 变毫秒级**(冷启动痛点顺手消解);
- 分享叙事升级:`uv tool install` + `cmem setup` 两条命令完成全部集成。

## 技术方案

### A. MCP server(新文件 `cmem/mcp_server.py`,entry point `cmem-mcp`)

官方 `mcp` SDK(FastMCP,stdio)。**工具克制为 3 个**(对照 MemPalace 的 36 个):

| 工具 | 参数 | 干什么 |
|---|---|---|
| `search_history` | query, k=5, before="", exclude_project="", source="" | 混合检索,可限定 claude/codex/cursor,返回块+出处;description 写明"被问过去的讨论/决策/原话时使用" |
| `get_session` | session_id, around=null | 从 text 当前投影取整场对话(或命中块前后文)——AI 检索后展开全景用 |
| `memory_status` | — | 库概况,让 AI 知道记忆覆盖范围 |

**常驻缓存与失效**:服务实例懒加载 embedder + (rows, matrix);每次查询前 stat
库文件 `(mtime_ns, size)`,变化(另一进程跑了 index)则重载矩阵。检索热路径不碰
raw(v0.3 已定原则)。

### B. `cmem setup`(集成一条命令)

- 默认:同时检测 `claude` / `codex` CLI 与 Cursor 安装/全局配置;
  前两者注册用户级 stdio server,Cursor 原子合并 `~/.cursor/mcp.json`;
  完成后跑 memory status 验活回显;
- `--instructions`:幂等更新 Claude 全局 CLAUDE.md + Codex 当前全局 AGENTS.md
  (`AGENTS.override.md` 非空时遵循官方优先级改写它),marker 包裹;
- `--claude-md`:保留为只写 Claude 文件的兼容模式;
- `--remove`:干净卸载(两种模式都撤);
- 写用户文件前回显将写入的内容;同名非 AICodeMemory 配置不覆盖、不移除。

### C. 顺手小件(v0.3 遗留)

- `cmem verify`:遍历 raw 逐份验 gzip CRC + 库 quick_check,报告损坏清单(478 份预计几秒);
- `cmem show <session-id前缀>`:CLI 版上下文展开(text 当前投影取整场对话),`--raw` 按需解压底片
  看未去噪原文(实测 60ms/份)。`get_session` 与它共享实现。

### D. README

- 安装节一条命令化:`uv tool install git+https://github.com/yangfurui/AICodeMemory`,
  附 uv → pipx → pip+venv 降级链(照顾无 uv 用户);
- 集成节改为:`cmem setup`(主推)/ `cmem setup --instructions`(轻量备选)双轨。

## 新增依赖

`mcp`(官方 SDK)。锁定稳定的 v1 系列 `mcp>=1.27,<2`;不引入其他。

## 验收标准

1. **真机闭环**:Claude/Codex/Cursor 客户端可见 cmem 三工具;
   从 Cursor 查询一条 Claude/Codex 历史,观察调用 `search_history`
   并引用原话回答,证明跨客户端价值;
2. **常驻性能**:server 内第二次查询(缓存热)延迟 < 200ms(对照 CLI 冷启动 3-4s);
3. **缓存失效**:server 存活期间另开终端跑 `cmem index` 索引新会话 → server 内再查能命中新内容;
4. `cmem setup` / `--instructions` / `--remove` 三态幂等可往返,
   `--claude-md` 兼容行为不回退;
5. `verify` 全绿(478 份);`show` 能展开任一会话,`--raw` 输出含被去噪的内容;
6. 契约测试与五连测不回退。
7. Cursor 普通 Agent 本地历史能进入 raw 与检索投影;重复索引无重复写入,
   上游数据库消失后仍能从 raw 重建;Background Agent 明确不在本地采集范围。
